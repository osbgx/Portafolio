## Resumen
Fui responsable del kick-off para la implementación del sistema hospitalario Anamnesis en el Sanatorio Fernández (CAMSA), liderando la transición desde un sistema heredado hacia una plataforma moderna basada en tecnologías Oracle, con Excelsis como partner tecnológico. El proyecto marcó el inicio del proceso de digitalización clínica y administrativa de la institución, estableciendo una base sólida para la estandarización de datos médicos y la modernización operativa.

Mi rol combinó liderazgo técnico y coordinación operativa: instalé y configuré el sistema, ejecuté la migración de datos, supervisé la detección de errores y coordiné acciones correctivas para garantizar estabilidad, integridad y continuidad durante la transición.

## Problemática
Antes de la implementación, el sanatorio enfrentaba desafíos significativos:

- Carencia de ciertos módulos esenciales para la gestión clínica y administrativa.
- Un sistema heredado con limitaciones para escalar y adaptarse a nuevas necesidades.
- Necesidad urgente de modernizar la infraestructura digital para soportar crecimiento futuro.

La dirección decidió adoptar Anamnesis como sistema hospitalario central, y fui asignado para liderar la fase inicial del proyecto.

## Solución
Mi trabajo consistió en establecer la base técnica y operativa para la adopción del sistema:

- Instalación y configuración del sistema hospitalario Anamnesis.
- Preparación del entorno Oracle.
- Migración parcial de datos desde la plataforma anterior.
- Carga masiva de información clínica y administrativa.
- Validación de integridad y consistencia de datos.
- Supervisión de errores y coordinación de correcciones.
- Alineación del sistema con los flujos operativos del sanatorio.

El objetivo fue asegurar un inicio estable, ordenado y sin interrupciones para las áreas médicas y administrativas.

## Arquitectura y datos
El sistema Anamnesis se basa en tecnologías Oracle, lo que permitió:

- Manejo robusto de datos clínicos.
- Escalabilidad para futuras ampliaciones.
- Integridad transaccional.
- Estandarización de estructuras médicas y administrativas.

Mis responsabilidades incluyeron:

- Preparación del entorno de base de datos.
- Ejecución de scripts de carga masiva.
- Validación de estructuras y relaciones.
- Corrección de inconsistencias heredadas del sistema anterior.
- Ajustes en PL/SQL para soportar la migración.

## Rol como Project Manager técnico
Además del trabajo técnico, asumí responsabilidades de coordinación:

- Supervisión del avance del proyecto.
- Comunicación con dirección y áreas médicas.
- Priorización de incidencias.
- Coordinación de pruebas funcionales.
- Aseguramiento de cumplimiento técnico y operativo.
- Planificación del despliegue inicial.

Esto permitió que la transición se realizara sin interrupciones en la atención médica.

## Impacto
La implementación inicial generó beneficios inmediatos:

- Base sólida para la digitalización clínica y administrativa.
- Estandarización de la captura de datos médicos.
- Reducción de errores derivados del sistema heredado.
- Mayor trazabilidad de información de pacientes.
- Infraestructura preparada para futuras fases del proyecto.
- Continuidad operativa durante la transición.

El trabajo realizado permitió que la institución avanzara hacia un ecosistema digital moderno y escalable.
